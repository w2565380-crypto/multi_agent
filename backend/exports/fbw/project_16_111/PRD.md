# 1. 项目概述
- **项目名称**: 彩虹计算器 (Rainbow Calculator)
- **业务目标**: 提供一个基础的四则运算计算功能，通过高饱和度、渐变色等丰富色彩设计提升视觉愉悦感，使简单工具更具吸引力。

# 2. 核心功能需求 (Functional Requirements)
## 2.1 计算器主界面
- **页面/界面说明**: 
  整个页面背景为深色（如#333）以凸显彩色元素。计算器主体为一个圆角矩形面板，包含：
  - 顶部显示区：分为表达式输入行（显示当前输入的数字或操作符，字体稍小）和结果输出行（显示计算结果，字体大而粗）。
  - 按钮区：4x4+1网格布局，包含数字0-9、小数点、加减乘除、等于、清除（C）、退格（←）。每个按钮使用不同的鲜艳色彩（如彩虹色环排序），按钮在悬停、按下时有颜色微变或缩放反馈。
- **组件与字段**:
  - `表达式显示区`（只读文本区域）: 显示当前输入的数字和运算符序列。
  - `结果显示区`（只读文本区域）: 显示计算结果，初始为“0”。
  - `数字按钮`（共10个）: 文本标签0-9，背景色分别取红(#FF0000)、橙(#FF7F00)、黄(#FFFF00)、绿(#00FF00)、青(#00FFFF)、蓝(#0000FF)、紫(#8B00FF)、粉(#FF69B4)、棕(#A0522D)、灰(#808080)。
  - `运算符按钮`（4个）: +、-、×、÷，背景色统一为蓝色系（如#1E90FF、#00BFFF等）。
  - `功能按钮`（2个）: C（清除）背景色红色(#FF4500)，⌫（退格）背景色橙色(#FF8C00)。
  - `等号按钮`（1个）: 背景色金色(#FFD700)，尺寸可略大以突出。
  - `小数点按钮`（1个）: 背景色白色(#FFFFFF)或浅灰(#D3D3D3)。
- **交互与业务逻辑**: 
  1. 初始状态：表达式显示区为空，结果显示区显示“0”。
  2. 用户点击数字按钮，该数字追加到表达式显示区（如果结果区有非“0”值，则先清除结果区，将其作为新输入的开始）。
  3. 用户点击运算符按钮：若当前表达式末尾为运算符，则替换该运算符；否则将运算符追加到表达式后，并清空输入状态等待下一个数字。
  4. 用户点击小数点：若当前输入数字中已包含小数点，则忽略；否则追加小数点。
  5. 用户点击等号：将表达式字符串通过JavaScript的eval或自定义解析器计算；若计算成功，结果显示区显示结果，表达式显示区保留原表达式；若出现语法错误或除以0，结果显示区显示“Error”，表达式区不变。
  6. 点击C：清除所有输入，表达式区清空，结果区恢复“0”。
  7. 点击⌫：删除表达式区最后一个字符（若结果区存在非“0”错误状态则重置）。
  8. 每个按钮在鼠标悬停时亮度提升10%，在按下时尺寸缩小至95%并加深颜色。
  9. 结果为正数时，结果区文字颜色为绿色；结果为负数时，结果为红色；结果为0时，显示为白色。

## 2.2 无其他模块（单页面应用）

# 3. 非功能性需求与约束
- **技术栈方向**: 纯HTML/CSS/JavaScript，单页面应用，无需后端支持。
- **颜色设计约束**: 
  - 按钮颜色须选自彩虹色系（红橙黄绿青蓝紫）并保证每个数字按钮颜色不同。
  - 整体色调明亮、饱和，避免暗沉或灰色调（除背景外）。
  - 色值示例：0=#FF0000, 1=#FF7F00, 2=#FFFF00, 3=#00FF00, 4=#00FFFF, 5=#0000FF, 6=#8B00FF, 7=#FF69B4, 8=#A0522D, 9=#808080。
  - 背景色建议#2C2C2C或深蓝#1a1a2e，按钮圆角8px，带轻微阴影。
- **交互限制**: 
  - 必须适配 Chrome、Firefox、Edge 等主流浏览器（支持CSS3和ES6）。
  - 响应式设计：在手机（最小宽度320px）和桌面（最大宽度600px）均能良好显示，按钮大小自适应（最小触控区域44x44px）。
  - 操作逻辑简单直观，符合一般计算器使用习惯，支持鼠标点击和触摸事件。
  - 按下按钮需有即时视觉反馈（CSS transition 0.1s）。
