# 1. 项目概述
- **项目名称**: 简易在线计算器
- **业务目标**: 为用户提供快速、准确的四则运算计算工具，界面直观、操作无门槛，无需安装即可在浏览器中使用。

# 2. 核心功能需求 (Functional Requirements)
## 2.1 数字与运算符输入
- **页面/界面说明**: 计算器主界面由三部分组成：顶部为数字/运算结果显示区，中间为运算符按钮区（+、-、×、÷、=、C），下方为数字按钮区（0-9、小数点）。按钮采用网格布局，每个按钮有清晰的文字标签。
- **组件与字段**:
  - `显示框`: 用于显示当前输入的数字、运算符及计算结果，默认显示“0”。
  - `数字按钮`: 0-9 共10个，每个按钮对应一个数字。
  - `小数点按钮`: “.”。
  - `运算符按钮`: “+”、“-”、“×”、“÷”。
  - `等于按钮`: “=”，触发计算。
  - `清除按钮`: “C”，将当前输入重置为“0”。
- **交互与业务逻辑**:
  1. 用户点击任意数字按钮，将该数字追加到显示框当前值的末尾（若当前值为“0”，则替换为点击的数字）。
  2. 用户点击小数点按钮，若当前值不包含小数点，则追加小数点；若当前值已包含小数点，则忽略该次点击。
  3. 用户点击运算符按钮，记录当前显示框的值作为第一个操作数，并记录该运算符，同时将显示框清空（或置为“0”）等待输入第二个操作数。
  4. 用户点击等于按钮时，执行已记录的运算，将结果显示在显示框中。若未输入第二个操作数，则显示原第一个操作数。
  5. 用户点击清除按钮，重置所有状态：清除当前输入、清除已记录的操作数和运算符，显示框恢复为“0”。

## 2.2 运算执行与结果显示
- **页面/界面说明**: 显示框位于计算器界面最上方，采用单行文本显示，字号较大。结果展示区域与输入区域为同一组件，无需额外界面。
- **组件与字段**:
  - `显示框`: 同2.1，用于展示运算过程和最终结果。
  - `等于按钮`: “=”，触发计算。
  - `错误提示`: 当出现除零等非法运算时，显示框显示“Error”。
- **交互与业务逻辑**:
  1. 当用户点击“=”时，系统获取已存储的第一个操作数、运算符和当前显示框的值（第二个操作数），执行对应运算（+、-、×、÷）。
  2. 若运算符为“÷”且第二个操作数为0，显示框展示“Error”，并清除所有记录状态（之后的输入需从“0”开始）。
  3. 计算结果精确到小数点后两位（普通模式），或保留浮点精度（简单模式可选）。本PRD选择保留浮点结果，不特意截断。
  4. 计算完成后，显示框显示结果，同时将结果作为下一次运算的第一个操作数（链式运算），运算符和第二个操作数记录清空。
  5. 若用户连续点击“=”，保持上一次运算结果不变（不重复运算）。

# 3. 非功能性需求与约束
- **技术栈方向**: HTML/CSS/JS（纯前端实现，无需后端服务）。
- **交互限制**:
  - 所有按钮点击均需提供即时视觉反馈（如按下变暗、松手恢复）。
  - 必须适配主流浏览器（Chrome、Firefox、Safari、Edge最新版本），响应式布局兼容移动端屏幕（最小宽度320px）。
  - 操作逻辑简单直观：所有操作仅通过鼠标/触摸点击完成，无需键盘输入，避免意外触发。
- **性能要求**: 按钮点击响应时间不超过200ms，计算执行时间不超过50ms。
