- **项目名称**: 简易计算器
- **业务目标**: 为用户提供一个快速、准确进行加、减、乘、除基本算术运算的工具，简化日常计算操作。

# 2. 核心功能需求 (Functional Requirements)
## 2.1 数字输入与结果显示
- **页面/界面说明**: 计算器界面顶部为一个显示区域，用于展示用户输入的表达式及最终计算结果；下方为数字按键区域。
- **组件与字段**:
  - `显示屏`: 显示当前输入的数字、运算符及计算结果（支持滚动显示较长表达式）。
  - `数字按钮`: 数字 0-9，小数点 “.”，正负切换按钮 “±”。
- **交互与业务逻辑**:
  1. 点击数字按钮时，对应数字追加到当前输入值末尾。
  2. 点击小数点按钮时，若当前输入中尚无小数点，则追加小数点；否则忽略。
  3. 点击 “±” 按钮时，将当前输入值取反（乘以 -1）。
  4. 每次输入变更时，实时刷新显示屏内容。

## 2.2 基本运算操作
- **页面/界面说明**: 数字按钮右侧/下方的运算操作区域，包含运算符按钮、等号按钮和清除按钮。
- **组件与字段**:
  - `运算符按钮`: 加 (+)、减 (-)、乘 (×)、除 (÷)。
  - `等号按钮`: = (计算结果)。
  - `清除按钮`: C (清除所有输入)、CE (清除当前输入)。
- **交互与业务逻辑**:
  1. 点击运算符按钮时，将当前输入值作为第一个操作数保存，并记录运算符，然后清空当前输入等待第二个操作数。
  2. 若连续点击多个运算符，则以最后一次点击为准（或保持上一次未完成操作状态）。
  3. 点击等号按钮时，使用已保存的第一个操作数、运算符及当前输入值进行计算，将结果显示在显示屏上，并清除保存的操作数和运算符。
  4. 除法时若除数为 0，则显示错误提示（如 “Error”）。
  5. 点击 “C” 按钮时，重置所有状态（操作数、运算符、当前输入均清零，显示屏清空）。
  6. 点击 “CE” 按钮时，仅清除当前输入值，不影响已保存的操作数和运算符。

# 3. 非功能性需求与约束
- **技术栈方向**: HTML/CSS/JS。
- **交互限制**: 必须适配主流浏览器（Chrome、Firefox、Edge、Safari），操作逻辑需简单直观，按钮大小适合手指触摸（适用于触屏设备）及鼠标点击。
