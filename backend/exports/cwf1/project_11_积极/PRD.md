# 1. 项目概述
- **项目名称**: 简单计算器
- **业务目标**: 提供基本的加减乘除四则运算功能，满足用户日常快速计算需求。

# 2. 核心功能需求 (Functional Requirements)
## 2.1 基本计算模块
- **页面/界面说明**: 一个简洁的图形用户界面，包含显示区域（文本框或屏幕）用于展示当前输入和计算结果，以及数字按键（0-9）、小数点、四则运算符（+、-、×、÷）、等号（=）按键。
- **组件与字段**:
  - `显示区域`: 用于显示输入的数字、运算符及结果。
  - `数字按钮`: 0-9 (10个按钮)
  - `小数点按钮`: “.” 
  - `运算符按钮`: “+”、“-”、“×”、“÷”
  - `等号按钮`: “=”
- **交互与业务逻辑**: 
  1. 用户点击数字按钮，数字追加到显示区域右侧；若显示区域当前显示“0”或刚显示结果，则替换为当前数字。
  2. 用户点击运算符按钮，记录当前输入数值和运算符，并清空显示区域等待第二个数字输入。
  3. 用户点击等号按钮，根据记录的运算符和两个数值执行计算，并在显示区域显示结果。
  4. 连续运算：结果显示后，继续点击运算符可基于结果进行下一步计算。
  5. 除法运算中若除数为0，显示错误提示“Error”。

## 2.2 辅助功能模块
- **页面/界面说明**: 在计算器界面上额外提供清除、退格、正负号切换按钮。
- **组件与字段**:
  - `清除按钮`: “C” 或 “AC”
  - `退格按钮`: “←”
  - `正负号按钮`: “+/−”
- **交互与业务逻辑**: 
  1. 点击清除按钮：重置所有输入，显示区域归零。
  2. 点击退格按钮：删除显示区域最后一个字符；若仅剩一位则显示“0”。
  3. 点击正负号按钮：将当前显示数值取反（乘以-1）。

# 3. 非功能性需求与约束
- **技术栈方向**: HTML/CSS/JavaScript 实现前端应用，所有运算在客户端完成，无需后端。
- **交互限制**: 界面必须简洁，按钮大小适中，适配常见屏幕尺寸（移动端和桌面端）。所有操作通过点击完成，无键盘输入限制，但建议同时支持键盘输入（如数字键和运算符键）以增强易用性。
- **性能要求**: 运算响应时间不超过 50ms，界面无卡顿。
