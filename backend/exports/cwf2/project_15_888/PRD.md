# 1. 项目概述
- **项目名称**: 简易桌面计算器
- **业务目标**: 为用户提供基础四则运算（加、减、乘、除）的即时计算能力，界面直观，操作反馈清晰，无需复杂配置即可完成日常简单数值运算。

# 2. 核心功能需求 (Functional Requirements)
## 2.1 计算器主界面与交互
- **页面/界面说明**:
  整体为一个固定尺寸的卡片式界面，背景为浅色（如白色/浅灰），包含顶部显示区（计算结果及输入表达式）和底部按键区（数字0-9、小数点、运算符+ - × ÷、清除C、等于=、退格⌫）。按键采用网格布局（4列），数字键为浅灰色，运算符、清除、等于键为不同颜色（运算符蓝色、清除红色、等于绿色）。显示区分为上下两行：上行显示当前输入的完整表达式（如“12+34”），下行显示当前输入的数字或最终结果。
- **组件与字段**:
  - `显示区域`: 包含两个文本行（上行表达式，下行结果/输入值），字号16px/24px，右对齐。
  - `数字按钮`: 0-9，共10个，点击后输入数字。
  - `小数点按钮`: “.”，输入小数点，同一数字内只能输入一次。
  - `运算符按钮`: “+”、“-”、“×”、“÷”，触发运算逻辑。
  - `等于按钮`: “=”，计算结果并显示。
  - `清除按钮`: “C”，重置所有状态，显示归零。
  - `退格按钮`: “⌫”，删除当前输入数字的最后一位，若数字仅一位则变为0。
- **交互与业务逻辑**:
  1. **数字输入**: 用户点击数字按钮，数字追加到当前输入（下行）；若当前状态为刚完成运算（已显示结果）或首次输入，则新数字覆盖显示。不允许以0开头连续输入（如“00”）。
  2. **小数点输入**: 点击小数点，若当前输入中已有小数点则忽略，否则追加小数点；若当前输入为空或刚显示结果，则自动补“0.”。
  3. **运算符输入**: 点击运算符（+ - × ÷，内部存储为+ - * /）：
     - 若当前有未完成的表达式（已输入第一个数字和运算符），则直接替换运算符。
     - 若当前已显示结果，则将结果作为第一个操作数，开始新运算。
     - 将当前输入（或结果）作为第一个操作数存储，记录运算符，清空输入区（下行），上行显示“第一个操作数 运算符”。
  4. **等于运算**: 点击“=”：
     - 若已有运算符和第二个操作数，则执行计算，结果显示在下行，上行显示完整表达式及结果（如“12+34=46”）。
     - 若缺少操作数（如仅输入了数字未按运算符），则显示当前数字，不上行显示。
     - 除法时若除数为0，下行显示“错误”（Error），上行保留表达式，后续输入需按C清除。
     - 结果保留10位小数，去除多余末尾零。
  5. **清除(C)**: 点击C，重置所有状态：上行清空，下行显示“0”，清除存储的操作数和运算符。
  6. **退格(⌫)**: 点击⌫，删除下行当前输入数字的最后一位字符；若删除后为空，显示“0”；若当前状态为错误或刚显示结果，按C处理（回到初始状态）。
  7. **错误处理**: 任何计算溢出（超出JS安全整数范围或除零）时，下行显示“错误”，上行保留操作状态，后续输入除C外均无效。

# 3. 非功能性需求与约束
- **技术栈方向**: 基于 HTML/CSS/JavaScript 的纯前端实现，无需服务端或数据库，单页面应用。或可选 Python 脚本（带 Tkinter 或命令行交互），本PRD优先采用Web版本。
- **交互限制**: 必须适配主流浏览器（Chrome、Firefox、Edge、Safari最新版本），响应式设计最小宽度320px；所有操作需即时反馈（点击按钮视觉变化0.15s，计算结果100ms内显示）。键盘快捷键：数字0-9、小数点、+-*/、Enter(=)、Backspace(退格)、Escape(C)。按钮点击需提供触控友好区域（至少44x44px）。
- **错误处理**: 除零、数值溢出、无效运算（如连续等于等）均需捕获并展示友好的“错误”提示，不会导致界面卡死。
