## 1. 获取项目成果预览 URL (GET)

### 1.1 接口描述
根据项目文件夹名称，动态获取该项目自动生成的网页（倒计时器/计算器等）的公网预览访问路径。前端拿到此路径后，可以直接通过 `<iframe>` 标签在页面内进行渲染。

### 1.2 请求信息
*   **请求方法 (Method)**: `GET`
*   **API 相对路径**: `/api/projects/{project_name}/preview-url`
*   **线上完整测试 URL 示例**: 
    ```text
    http://8.160.185.36:8000/api/projects/my_first_project/preview-url
    ```

### 1.3 请求参数 (Request Parameters)
#### 路径参数 (Path Parameters)
| 参数名         | 类型     | 是否必填 | 描述                          | 示例值             |
| :------------- | :------- | :------- | :---------------------------- | :----------------- |
| `project_name` | `string` | 是       | Dify 生成的本地项目文件夹名称 | `my_first_project` |

---

### 1.4 返回参数说明 (Response Fields)
| 字段名                | 类型      | 描述                                                          |
| :-------------------- | :-------- | :------------------------------------------------------------ |
| `success`             | `boolean` | 接口执行状态，`true` 代表成功，`false` 代表失败               |
| `project_name`        | `string`  | 当前查询的项目文件夹名称                                      |
| `preview_url`         | `string`  | **核心返回字段**。用于前端 `<iframe>` 渲染的完整网页 URL 地址 |
| `debug_resolved_path` | `string`  | 后端在服务器上实际匹配的物理绝对路径（供前后端联调测试使用）  |

---

### 1.5 返回示例 (Response Examples)

#### 成功返回 (Status Code: 200)
```json
{
  "success": true,
  "project_name": "my_first_project",
  "preview_url": "http://8.160.185.36:8000/previews/my_first_project/src/index.html",
  "debug_resolved_path": "/home/ai_agent_company/backend/exports/my_first_project"
}
```

#### 失败返回 (Status Code: 404 - 项目未生成或路径不存在)
```json
{
  "detail": "项目 'non_existent_project' 文件夹不存在，物理路径: /home/ai_agent_company/backend/exports/non_existent_project"
}
```





























# 🚀 AI 多智能体模拟系统 - 前端对接接口文档

本文档定义了系统用户管理与认证模块的 API 规范。前端（Vue / Streamlit 等）在联调时请严格按照对应的请求格式与字段名称进行调用。

## 📌 全局配置与规范

* **后端 Base URL**: `http://127.0.0.1:8000`
* **内容类型 (Content-Type)**: `application/json`
* **在线交互文档 (Swagger UI)**: 后端启动后，前端可直接访问 `http://127.0.0.1:8000/docs` 进行可视化接口调试。
* **跨域支持 (CORS)**: 后端已全量开启跨域允许，前端可直接发起异步请求。

---

## 🔐 1. 用户认证模块

### 1.1 用户注册（创建用户）

* **请求路径**: `/api/users`
* **请求方法**: `POST`
* **功能描述**: 注册并向 SQLite 数据库中录入一个全新用户，系统会自动检测用户名是否重复。

#### 📥 请求参数 (Body - JSON)

| 字段名      | 类型   | 必填 | 约束限制   | 含义说明                     |
| ----------- | ------ | ---- | ---------- | ---------------------------- |
| `user_name` | String | 是   | 2-50个字符 | 用户名（全库唯一，不可重复） |
| `password`  | String | 是   | 至少1位    | 登录密码（直接传入明文）     |

* **请求体示例**:

```json
{
  "user_name": "fbw",
  "password": "my_password_123"
}

```

#### 📤 响应结果 (Response - JSON)

* **状态码**: `201 Created`
* **成功返回示例**:

```json
{
  "id": 1,
  "user_name": "fbw",
  "password": "my_password_123"
}

```

#### ❌ 异常状态码

* `400 Bad Request`: 用户名已被占用。返回体格式：`{"detail": "用户名已被占用"}`
* `422 Unprocessable Entity`: 参数校验不通过（如用户名长度不足、字段缺失）。

---

### 1.2 用户登录校验

* **请求路径**: `/api/users/login`
* **请求方法**: `POST`
* **功能描述**: 提供明文比对的登录校验。登录成功后前端应在本地缓存（如 `localStorage`）中记录 `user_id`。

#### 📥 请求参数 (Body - JSON)

| 字段名      | 类型   | 必填 | 含义说明   |
| ----------- | ------ | ---- | ---------- |
| `user_name` | String | 是   | 登录用户名 |
| `password`  | String | 是   | 登录密码   |

* **请求体示例**:

```json
{
  "user_name": "wzp1",
  "password": "123456"
}

```

#### 📤 响应结果 (Response - JSON)

* **状态码**: `200 OK`
* **成功返回示例**:

```json
{
  "status": "success",
  "user_id": 2,
  "user_name": "wzp1"
}

```

#### ❌ 异常状态码

* `401 Unauthorized`: 用户名或密码错误。返回体格式：`{"detail": "用户名或密码错误"}`

---

## 👥 2. 用户管理基础功能 (CRUD)

### 2.1 获取所有用户列表

* **请求路径**: `/api/users`
* **请求方法**: `GET`
* **功能描述**: 获取当前系统中注册的所有用户数据，主要用于测试或管理员面板。

#### 📥 请求参数

无

#### 📤 响应结果 (Response - JSON)

* **状态码**: `200 OK`
* **成功返回示例**:

```json
[
  {
    "id": 1,
    "user_name": "fbw",
    "password": "my_password_123"
  },
  {
    "id": 2,
    "user_name": "wzp1",
    "password": "123456"
  }
]

```

---

### 2.2 根据 ID 获取指定用户

* **请求路径**: `/api/users/{user_id}`
* **请求方法**: `GET`
* **功能描述**: 传入用户主键 ID，获取该用户的详细信息。

#### 📥 路径参数 (Path)

| 参数名    | 类型    | 必填 | 含义说明          |
| --------- | ------- | ---- | ----------------- |
| `user_id` | Integer | 是   | 目标用户的唯一 ID |

#### 📤 响应结果 (Response - JSON)

* **状态码**: `200 OK`
* **成功返回示例**:

```json
{
  "id": 1,
  "user_name": "fbw",
  "password": "my_password_123"
}

```

#### ❌ 异常状态码

* `404 Not Found`: 该用户 ID 在数据库中不存在。返回体格式：`{"detail": "用户不存在"}`

---

### 2.3 修改用户密码

* **请求路径**: `/api/users/{user_id}`
* **请求方法**: `PUT`
* **功能描述**: 允许修改指定用户 ID 的用户名和密码。

#### 📥 路径参数 (Path)

| 参数名    | 类型    | 必填 | 含义说明          |
| --------- | ------- | ---- | ----------------- |
| `user_id` | Integer | 是   | 目标用户的唯一 ID |

#### 📥 请求参数 (Body - JSON)

| 字段名      | 类型   | 必填 | 含义说明       |
| ----------- | ------ | ---- | -------------- |
| `password`  | String | 是   | 修改后的新密码 |
| `user_name` | String | 是   | 修改后的用户名 |


* **请求体示例**:

```json
{
  "user_name": "new_fbw",
  "password": "new_secure_password"
}

```

#### 📤 响应结果 (Response - JSON)

* **状态码**: `200 OK`
* **成功返回示例**:

```json
{
  "id": 1,
  "user_name": "fbw",
  "password": "new_secure_password"
}

```

#### ❌ 异常状态码

* `404 Not Found`: 用户不存在。

---

### 2.4 删除用户

* **请求路径**: `/api/users/{user_id}`
* **请求方法**: `DELETE`
* **功能描述**: 注销/物理删除指定用户。由于后端数据库配置了 `ON DELETE CASCADE` 级联删除机制，**删除该用户会将其名下的所有协同仿真项目、历史日志一并安全清空**，前端调用时请做二次确认弹窗。

#### 📥 路径参数 (Path)

| 参数名    | 类型    | 必填 | 含义说明            |
| --------- | ------- | ---- | ------------------- |
| `user_id` | Integer | 是   | 被删除用户的唯一 ID |

#### 📤 响应结果

* **状态码**: `204 No Content`（执行成功，无任何内容返回）

#### ❌ 异常状态码

* `404 Not Found`: 用户不存在。

---

## 💡 前端避坑指引

1. **字段严禁大小写混淆**：后端的参数字段采用的是下划线命名法（如 `user_name`），请勿在前端写成驼峰式的 `userName`，否则触发 `422` 校验不通过错误。
2. **状态码处理**：请拦截 `400` 和 `401` 状态码，通过 `error.response.data.detail` 获取后端抛出的具体中文字符串并使用 `Message/Notification` 组件弹出提示。

# 🚀 AI 多智能体模拟系统 - 前端对接接口文档 (v1.3.0)

本文档定义了系统用户管理、认证以及头像上传模块的最新 API 规范。

## 📌 全局配置与规范

* **后端 Base URL**: `http://127.0.0.1:8000`
* **内容类型 (Content-Type)**: `application/json` (文件上传接口除外)
* **用户头像网络根路径**: `http://127.0.0.1:8000/avatars/`
* **在线交互文档 (Swagger UI)**: `http://127.0.0.1:8000/docs`

---

## 🔐 1. 用户认证与基础管理模块

### 1.1 用户注册（创建用户）

* **请求路径**: `/api/users`
* **请求方法**: `POST`
* **功能描述**: 注册全新用户，初始状态下用户的头像属性 `image` 为 `null`。

#### 📥 请求参数 (Body - JSON)

| 字段名      | 类型   | 必填 | 约束限制   | 含义说明           |
| ----------- | ------ | ---- | ---------- | ------------------ |
| `user_name` | String | 是   | 2-50个字符 | 用户名（全库唯一） |
| `password`  | String | 是   | 至少1位    | 登录密码（明文）   |

* **请求体示例**:

```json
{
  "user_name": "fbw",
  "password": "my_password_123"
}

```

#### 📤 响应结果 (Response - JSON)

```json
{
  "id": 1,
  "user_name": "fbw",
  "password": "my_password_123",
  "image": null
}

```

---

### 1.2 用户登录校验

* **请求路径**: `/api/users/login`
* **请求方法**: `POST`
* **功能描述**: 明文比对登录。成功后会同步返回该用户的头像 `image` 网络绝对路径（若未上传则为 `null`）。

#### 📥 请求参数 (Body - JSON)

```json
{
  "user_name": "wzp1",
  "password": "123456"
}

```

#### 📤 响应结果 (Response - JSON)

```json
{
  "status": "success",
  "user_id": 2,
  "user_name": "wzp1",
  "image": "http://127.0.0.1:8000/avatars/user_2_avatar.jpg"
}

```

---

### 1.3 修改用户信息（支持改用户名、密码、头像）

* **请求路径**: `/api/users/{user_id}`
* **请求方法**: `PUT`
* **功能描述**: 允许用户动态修改自己的信息。所有参数均为**可选**，传哪个字段就修改哪个字段。
* **重名保护**: 如果传入了新的 `user_name` 且已被他人占用，后端将直接拦截并返回 `400` 错误。

#### 📥 路径参数 (Path)

| 参数名    | 类型    | 必填 | 含义说明          |
| --------- | ------- | ---- | ----------------- |
| `user_id` | Integer | 是   | 目标用户的唯一 ID |

#### 📥 请求参数 (Body - JSON)

| 字段名      | 类型   | 必填 | 含义说明                                                 |
| ----------- | ------ | ---- | -------------------------------------------------------- |
| `user_name` | String | 否   | 变更后的全新用户名                                       |
| `password`  | String | 否   | 变更后的全新密码                                         |
| `image`     | String | 否   | 头像文件名（通常由头像上传接口自动维护，无需前端手动改） |

* **请求体示例（仅修改用户名）**:

```json
{
  "user_name": "fbw_new_name"
}

```

#### 📤 响应结果 (Response - JSON)

```json
{
  "id": 1,
  "user_name": "fbw_new_name",
  "password": "my_password_123",
  "image": "http://127.0.0.1:8000/avatars/user_1_avatar.jpg"
}

```

---

### 1.4 获取所有用户列表 / 根据 ID 获取指定用户

* **请求路径**: `/api/users` (列表) 或 `/api/users/{user_id}` (单用户)
* **请求方法**: `GET`
* **功能描述**: 返回的用户数据中，`image` 字段已由后端自动拼装为**可直接用于 `<img>` 标签渲染的绝对网络 URL 链接**。

#### 📤 响应结果示例 (单用户)

```json
{
  "id": 1,
  "user_name": "fbw",
  "password": "my_password_123",
  "image": "http://127.0.0.1:8000/avatars/user_1_avatar.jpg"
}

```

---

### 1.5 删除用户

* **请求路径**: `/api/users/{user_id}`
* **请求方法**: `DELETE`
* **安全警告**: 删除该用户会通过 SQLite 的外键级联机制，**将其名下的所有多智能体仿真项目一并清空**。

#### 📤 响应结果

* **状态码**: `204 No Content`

---

## 📷 2. 物理文件上传模块

### 2.1 上传/更换用户头像

* **请求路径**: `/api/users/{user_id}/upload-avatar`
* **请求方法**: `POST`
* **内容类型 (⚠️注意)**: `multipart/form-data`
* **功能描述**: 前端通过表单形式上传图片文件，后端会将其重命名并安全保存至本地 `backend/data/images/` 目录下，并更新该用户的数据库记录。

#### 📥 请求参数 (Form Data)

| 参数名 | 类型          | 是否必填 | 含义说明                                  |
| ------ | ------------- | -------- | ----------------------------------------- |
| `file` | Binary (File) | 是       | 物理图片文件（限制必须为 `image/*` 格式） |

#### 📤 响应结果 (Response - JSON)

* **状态码**: `200 OK`
* **成功返回示例**:

```json
{
  "status": "success",
  "message": "头像上传成功",
  "image_url": "http://127.0.0.1:8000/avatars/user_1_avatar.jpg"
}

```

---

## ❌ 全局通用异常错误码对照表

| 状态码 | 触发场景                   | 后端返回的 `detail` 文本提示                             |
| ------ | -------------------------- | -------------------------------------------------------- |
| `400`  | 注册/修改用户名冲突        | `"用户名已被占用"` 或 `"该用户名已被其他人占用，请更换"` |
| `400`  | 上传非图片格式文件         | `"只能上传图片文件"`                                     |
| `401`  | 登录账号或密码输入错误     | `"用户名或密码错误"`                                     |
| `404`  | 路径中的 `user_id` 不存在  | `"用户不存在"`                                           |
| `422`  | 前端字段名写错或缺失必填项 | *(由 Pydantic 自动生成的字段缺失或类型错误报告)*         |

---

## 💡 前端联调指南

1. **图像直接渲染**: 无论是通过 `GET`、`PUT` 还是 `Login` 接口，只要返回的 `image` 字段不为 `null`，它就是一个完整的 URL 地址，前端直接将其赋值给 `<img :src="user.image" />` 即可。
2. **修改重名处理**: 前端在执行修改用户名操作时，如果收到 `400` 状态码，应通过 `error.response.data.detail` 提取中文字符串并弹窗提醒用户“名字已被占用”。
